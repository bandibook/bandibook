## MVC 패턴

- 애플리케이션의 구조를 개선하기 위해
- **관심사의 분리를 활용**

```
💡 비즈니스 데이터(모델), UI(뷰), 로직과 사용자 입력을 관리(컨트롤러)로 나눈다.
```

- 모델(Model)
  - 도메인 관련 데이터
  - UI에 대해서는 관여 X
- 뷰(View)
  - 모델의 현재 상태를 표현
  - 화면에 보여지는 부분만 담당
- 컨트롤러(Controller)
  - 키보드 입력이나 클릭 같은 사용자의 상호작용을 처리
  - 뷰에 무엇을 보여줄지, 사용자 입력을 어떻게 처리할지 등을 결정하는 역할
