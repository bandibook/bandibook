> 렌더링 패턴은 사용 사례에 맞는 콘텐츠 렌더링 방식을 제공

1. 클라이언트 사이드 렌더링: HTML이 클라이언트에서 완전히 렌더링
2. 서버 사이드 렌더링: HTML 콘텐츠를 서버에서 동적으로 렌더링한 후, 클라이언트에서 하이드레이션
3. 정적 렌더링: 빌드 타임에 서버에서 페이지를 렌더링하여 정적인 사이트를 구축
4. 점진적 정적 생성: 초기 빌드 이후에도 정적 사이트를 동적으로 추가하거나 수정 가능
5. 스트리밍 SSR: 서버에서 렌더링된 콘텐츠를 더 작은 스트림 조각으로 분할하여 전송
6. 엣지 렌더링: 렌더링된 HTML을 클라이언트에 전송하기 전에 엣지에서 수정
7. 하이브리드 렌더링: 빌드 타임, 서버 및 클라이언트 렌더링을 결합하여 웹 개발에 더 유연한 접근 방식을 제공(ex. 리액트 서버 컴포넌트 및 Next.js의 App Router)
8. 부분 하이드레이션: 클라이언트에서 일부 컴포넌트만 하이드레이션
9. 점진적 하이드레이션: 클라이언트에서 컴포넌트 하이드레이션 순서를 제어
10. 아일랜드 아키텍처: 정적인 사이트 내에 여러 진입점을 가진 동적인 동작의 격리된 영역을 만든다.
11. 점진적 향상: 자바스크립트 없이도 애플리케이션이 작동하도록 보장

다음과 같은 근본적인 질문에 대한 해답을 찾을 수 있을 것도

- 콘텐츠를 어디에서 렌더링할 것인가?
- 웹 서버, 빌드 서버, 엣지 네트워크 또는 클라이언트, 어디에서 콘텐츠를 렌더링할 것인가?
- 콘텐츠를 한 번에, 부분적으로, 또는 점진적으로 어떻게 렌더링할 것인가?
